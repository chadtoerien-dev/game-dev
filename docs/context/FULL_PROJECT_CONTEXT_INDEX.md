## File coverage

Current cumulative source is unpacked in the repository. Historical package contents are represented by current source, versioned milestone documents and the implementation history. The exact generated ZIPs are recorded by hash in the artifact manifest and are available in the verified portable 0.6 handoff archive.

The GitHub connector did not preserve a test binary byte-for-byte, so the unverified duplicate was deliberately removed rather than merged. This does not remove project source or context from the repository.

## Preservation approach

User-authored master prompts are preserved verbatim. Major assistant outputs are preserved canonically across dedicated documents and current source. The message ledger identifies whether each turn is verbatim, canonically reconstructed, summarised from earlier-thread context or represented by milestone history.

## Deliberate exclusions

The public archive excludes hidden reasoning, platform instructions, tool-internal payloads, unrelated personal data, account details, copyrighted reference-game assets and unlicensed music. These are not project deliverables and must not be committed to a public repository.
